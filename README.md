## 🧠 Overview

**GitGuard AI** is a web-browsing AI agent trained to act like a senior software engineer managing a busy open-source repository. Instead of operating via direct API calls, the agent **visually navigates a custom-built mock GitHub interface**.

It reads issue descriptions and code diffs, then performs actions by executing BrowserGym UI commands — exactly the way a human maintainer would.

The agent uses **GRPO** to learn optimal navigation and triage strategies purely from numeric reward signals generated by an automated backend grader, requiring **zero human feedback labels**.

---

## 🌍 Environment Description

The mock environment is a fully functional web application simulating a GitHub-like repository management system.

**Frontend (The "Eyes")**
A React + Vite application featuring an Issues Inbox, Pull Request diff viewers, label dropdowns, and comment modules.

**Backend (The "Grader")**
A FastAPI server backed by SQLite exposing endpoints to:

- Reset the environment with randomized mock data — `POST /api/reset`
- Process agent interactions — `POST /api/update`
- Calculate GRPO reward scores based on the resulting database state — `GET /api/grade`

---

## 🔭 Observation & Action Spaces

### Observation Space — Accessibility Tree (`axtree_txt`)

The agent does **not** see raw pixels. Instead, the environment returns a structured **Accessibility Tree** — a clean, text-based DOM representation of the current React UI, including interactive elements and their unique IDs.

```
[Button] "Bug" (id=12) | [Textbox] "Linked Issue" (id=15)
```

### Action Space — BrowserGym Python Commands

The agent outputs specific Python-based browser commands to interact with the parsed UI.

| Command                   | Description                  |
| ------------------------- | ---------------------------- |
| `click("12")`             | Clicks the Bug label button  |
| `fill("15", "Issue #42")` | Types into a text box        |
| `noop()`                  | Waits for the page to render |

## 🎯 The Three Challenges

| Competency              | Task                                                                                | Difficulty | Grader Condition                                                     |
| ----------------------- | ----------------------------------------------------------------------------------- | ---------- | -------------------------------------------------------------------- |
| **Text Classification** | Read an issue and apply the correct label (`bug`, `docs`, `enhancement`)            | 🟢 Easy    | DB `status` matches the expected label                               |
| **Semantic Reasoning**  | Detect a duplicate issue and close it while linking the original issue ID           | 🟡 Medium  | DB `status` is `closed` and `linked_issue` matches original ID       |
| **Code Understanding**  | Spot logic bugs in a PR (e.g., off-by-one errors) and request changes with comments | 🔴 Hard    | DB `status` is `changes_requested` with text in the `comments` field |

---

## 🏆 Reward Function

The environment relies on a **sparse and partial reward structure** to guide the GRPO algorithm.

| Environment State / Action                             | Reward Signal |
| ------------------------------------------------------ | ------------- |
| Successfully resolving an Easy / Medium / Hard task    | **+1.0**      |
| Execution error / Invalid BrowserGym command           | **−0.5**      |
| General fallback / Task incomplete                     | **−0.1**      |
| ⚠️ Critical Failure: Merging a PR with an injected bug | **−2.0**      |

## ⚙️ Setup & Installation (Local)

To run the mock environment locally for testing or as the target for your training agent:

**1. Clone the repository and install dependencies**

```bash
git clone https://github.com/your-username/gitguard-ai.git
cd gitguard-ai
pip install -r requirements.txt
```

**2. Start the FastAPI backend (the grader)**

```bash
# In terminal 1
uvicorn environment.main:app --host 0.0.0.0 --port 8000
```

**3. Start the React frontend**

```bash
# In terminal 2
cd environment/frontend
npm install
npm run dev
```

## 🚀 Training the Agent (Google Colab)

Because training `Qwen2.5-1.5B-Instruct` requires significant VRAM, the GRPO loop is designed to run on a **Google Colab T4 GPU**, connecting to your local environment via a secure tunnel.

**1. Expose your local environment**

Create an SSH tunnel to expose your local React frontend to the public internet:

```bash
# In terminal 3
ssh -o ServerAliveInterval=60 -R 80:localhost:5173 nokey@localhost.run
```

> Copy the generated `https://...lhr.life` URL and open it once in your browser to clear the security warning.

**2. Set up Colab**

Upload the `agent/` folder to a Google Colab instance with a **T4 GPU** enabled. Install dependencies in isolated steps to prevent memory crashes:

```python
!pip install torch transformers datasets requests
!pip install trl
!pip install browsergym playwright
!playwright install chromium
```

**3. Run GRPO training**

Open `agent/train.py` in Colab, update `TUNNEL_URL` with your active `localhost.run` link, then run:

```bash
!cd agent && python train.py
```

> The script is optimized for T4 GPUs using `torch.float16` and synced `generation_batch_size` parameters.
